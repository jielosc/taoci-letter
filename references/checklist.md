# 生成后自检清单

写完信后，逐条对照。任何一条没过，重写对应段落。

## 格式（标准结构）
- [ ] 主题行含：申请类别 + 姓名 + 院校专业 + 排名/GPA + 意向方向
- [ ] 称呼用"尊敬的X老师/教授："，不是"您好"开头
- [ ] 首段直接交代院校/专业/年级/排名/升学途径，无寒暄
- [ ] 核心优势（论文、排名）已轻微加粗
- [ ] 有规范落款和祝语
- [ ] 提到附件转 PDF、规范命名、不发压缩包（作为给用户的提醒，不一定写进信里）

## 内核（top 平台加分项，比格式更重要）

**真诚** —— 能不能删掉客套话？
- [ ] 没有"久仰大名""非常向往贵组"这类空话
- [ ] 具体讲到导师近几年做的某个问题/某篇论文，并说出自己的一点理解
- [ ] 不是一上来就罗列成绩堆奖项

**切题** —— 这封信能不能原样发给另一个导师？
- [ ] 第二段的内容是这位导师专属的，换个人就不成立
- [ ] 体现出"翻过导师近几年论文"，而非"对您方向很感兴趣"的泛话
- [ ] 进阶：结合领域进展，提到一个导师接下来可能关心的问题

**潜力** —— 让导师看到的是"理解力"而非"履历厚度"
- [ ] 讲项目时体现了对问题"为什么值得做"的理解，不是流水账
- [ ] 没有密密麻麻堆竞赛/奖项却让人不知道你想做什么
- [ ] 整封信看完，导师能答出：①你了解他在做什么 ②你想做的题和他一致 ③你有继续科研的理解力和成长空间

## 最后一问
把信通读一遍，问自己：**这读起来像一个真实、有判断力的人写的，还是像 AI 套模板？** 如果像后者，回到"切题"重写第二段。
