# 套磁信生成器

帮保研/申研的同学写套磁信。导师每年收几十封信，绝大多数同质化、像 AI 套模板。本 skill 的目标不是写一封"语法正确、格式漂亮"的信，而是让导师看到邮件背后那个真实的人。

核心信条（贯穿全程，比格式更重要）：
- **真诚**：真的花时间看过导师研究，能讲出具体问题和自己的理解，而不是"久仰大名""非常向往贵组"。不要一上来罗列成绩堆奖项。
- **切题**：翻过导师近几年论文，结合领域新进展，能猜到导师接下来可能关心什么问题——最好正中导师一直想做的题。
- **潜力**：展示对自己做过的工作的真正理解和判断力，而不是把自己包装得"很厉害"。

## 工作流程

### 第 1 步：确认输入
**学生背景优先从 `profile.md` 读取**（与本 SKILL.md 同目录）。流程：
1. 先用 Read 读取本 skill 目录下的 `profile.md`。
2. 若文件存在且已填写（不是模板占位符），直接用其中的背景，**不要再问用户**。
3. 若 `profile.md` 不存在，提示用户：`cp profile.example.md profile.md` 后填写，今后即可免输入；本次可让用户在对话里直接给背景。
4. 若用户本次在对话里临时给了不同背景，以对话输入为准。

每次仍需用户提供的，只有：
- **导师主页链接**（学校官网 / 实验室主页 / Google Scholar 均可）

profile.md 里应包含：学校+专业+年级、GPA/排名、科研经历（项目/论文/方法）、技能栈、申请方向、其他亮点、姓名、信件语言偏好、申请类别。

### 第 2 步：抓取并分析导师主页
用 WebFetch 读链接，提取：导师**自己措辞**的研究方向、**近 2-3 年代表作**（标题 + 发表年份 + 能拿到的链接）、课题组/招生信息、任何能证明"我认真看过"的细节。
主页信息太少时，用 WebSearch 搜"导师姓名 + 单位 + research"补充，或请用户补一个 Scholar/实验室链接。

**主页只给标题，绝不够。** 主页上的一句话介绍不能作为写信依据——只凭标题臆测论文内容，是套磁信最常见的硬伤，导师一眼就能看出你没真读过。所以必须进入下一步。

**导师姓名必须从主页原文核对，绝不凭印象或链接里的拼音猜测：**
1. 从主页正文里找到导师的**全名**和**姓氏**，原样记下来（含正确汉字 / 英文拼写）。
2. 中文名注意：复姓、生僻字、易错同音字都要对准主页写法；英文名注意姓与名的顺序。
3. 主页上同时出现多个名字（同事、合作者、学生）时，确认哪个才是收信人本人。
4. 若主页没有明确姓名或存在歧义，**停下来问用户**，不要先写信。
5. 写信时称呼、正文中出现的导师姓名，全部用这一步核对过的写法，写完后再比对一次。

### 第 3 步：精读最相关的 1-2 篇论文（不可跳过）
匹配信只能由真正读过的论文支撑。光看主页标题就动笔，是这类信最常见的失败原因。流程：
1. 从第 2 步的代表作里，挑出**与学生背景最可能产生交集的 1-2 篇**（不是最新、不是引用最高，而是最切题的）。
2. 对每篇逐一拿到原文内容再读，按可得性依次尝试：
   - 用 WebSearch 搜"论文标题"找到 arXiv / 期刊 / 项目页链接；
   - 用 WebFetch 打开该链接，**至少读到 abstract + introduction + 方法/结论**，而不是只看标题或他人转述的一句话。
3. 每篇论文落实三件事，记下来：①它到底要解决什么问题、为什么这个问题重要；②用了什么核心方法/思路；③局限或留下的开放问题。
4. **拿不到全文时（付费墙、链接失效），明确承认只读到摘要**，匹配点措辞相应收敛，绝不就标题脑补论文细节当成读过的样子写进信里。
5. 读完若发现某篇与学生其实不契合，果断换一篇，宁可少写一个匹配点，也不硬凑。

### 第 4 步：做匹配并先给用户看（最关键）
逐条比对学生背景与导师研究，找 2-3 个**真实、具体**的交集（做过的项目↔导师方向、掌握的方法↔导师论文技术、兴趣↔导师近期具体问题）。匹配点必须落在第 3 步真正读过的论文内容上，而不是主页上的方向标签。
进阶：尝试基于导师论文+领域趋势，提出一个"导师接下来可能关心的问题"的想法。
**先把匹配点和这个想法列给用户确认，再写信。** 每个匹配点旁注明它来自哪篇论文、读到了哪一层（全文/仅摘要）。宁可承认"无直接经验但有兴趣"，绝不编造。

### 第 5 步：按七段结构生成
按 references/structure.md 写信。

**写短靠结构，不靠硬砍字数。** 长信的根源通常是重复和注水，不是内容多：主题行已交代的客观信息（姓名/院校/排名/方向），各段一句带过、绝不复述第二遍；每段只做自己那一件事；把篇幅集中到第二段（学术契合）。删掉所有"换个导师也成立"的客套后，正文自然会落在一屏以内（约 600 字是结果，不是去凑的指标）。

**用 Write 生成一份 markdown 文件交付，文件名 `套磁信-{导师姓名}.md`**（导师姓名用第 2 步核对过的写法）：
- 信件正文正常用 markdown，把核心优势（高水平论文、专业前列排名）用 `**加粗**` 标出来，便于导师一眼扫到。
- 加粗只用在真正的核心优势上，别滥用；其余靠措辞和语序前置。
- 不要在终端回复里再贴一遍全文，交付物就是这份 .md 文件。

**必须提醒用户正确的复制姿势**（否则加粗照样失效）：
> 请用 markdown 预览打开这份文件（VS Code 里按 `Cmd+K V`，或用 Typora），从**渲染后的预览视图**里选中复制，再粘到邮箱的富文本编辑框。直接从文件源码复制会带着 `**` 符号，加粗会失效。

### 第 6 步：举证式自检（只查内核，不刷绿勾）
格式项（主题行、称呼、落款）写的时候就在遵守，事后逐条打 ✅ 没有意义、纯浪费。自检只做真正能拦住烂信、且无法靠打勾蒙混的内核检查——**逐条举证，不准只写"✅通过"**：

1. **切题举证**：贴出第二段里谈论文的那句原话，紧跟着写出它依据的论文细节（解决什么问题/什么方法），并标明读到哪一层（全文/仅摘要）。**举不出对应细节，或细节其实来自主页标题脑补 → 判定没过**，回第 3 步补读论文、重写第二段。
2. **真诚检查**：通读全信，找出任何一句"换个导师也成立"的客套或泛话（如"久仰大名""对您方向很感兴趣"），列出来删改。找不到可删的才算过。
3. **潜力检查**：确认讲项目时写的是"为什么值得做"的理解，不是奖项/竞赛流水账。是流水账 → 重写第五段。
4. **冗余检查**：找出全信重复出现的客观信息（院校/排名/方向在主题行外又复述）和可删的注水句，列出来删掉。删干净后通常一屏读完；若仍明显偏长，说明某段没收住，回去精简而不是硬截字数。

### 第 7 步：独立子 agent 终审（对抗性，不可自评代替）
主 agent 是这封信的作者，自评天然偏宽。最后一关交给一个**全新上下文的子 agent**，它没参与写作、不维护"自己的作品"，最接近真实导师那种挑剔旁观视角。用 Task/Agent 工具起一个子 agent：

**喂给它**（缺一不可）：①套磁信全文；②第 3 步的论文精读笔记或论文链接——没有这个，子 agent 没读过论文，判断不了"论文理解有没有偏差"，这步就白做；③学生背景要点。

**让它扮演收信的导师，只挑毛病、不夸奖，返回结构化结论**：
- 信里对论文的每一处描述，对照笔记是否准确？有没有把方法/结论说错、或拿主页标题脑补当读过？
- 第二段换个导师还成不成立？哪句是"对您方向很感兴趣"式泛话？
- 通读后最直接的判断：像真人还是像 AI 套模板？
- 给出 `通过 / 需重写` 的明确结论 + 具体问题清单。

子 agent 判 `需重写` → 主 agent 按它列的问题改，必要时回第 3 步补读论文，改完可再审一轮。
**只向用户汇报：子 agent 发现了什么、改了什么**；若一次通过，一句"独立终审通过"即可，不贴长篇评审。
