# 套磁信生成器

帮保研/申研的同学写套磁信。导师每年收几十封信，绝大多数同质化、像 AI 套模板。本 skill 的目标不是写一封"语法正确、格式漂亮"的信，而是让导师看到邮件背后那个真实的人。

核心信条（贯穿全程，比格式更重要）：
- **真诚**：真的花时间看过导师研究，能讲出具体问题和自己的理解，而不是"久仰大名""非常向往贵组"。不要一上来罗列成绩堆奖项。
- **切题**：翻过导师近几年论文，结合领域新进展，能猜到导师接下来可能关心什么问题——最好正中导师一直想做的题。
- **潜力**：展示对自己做过的工作的真正理解和判断力，而不是把自己包装得"很厉害"。

## 工作流程

### 第 1 步：确认输入
**学生背景优先从 `profile.md` 读取**（与本 SKILL.md 同目录）。流程：
1. 先用 Read 读取本 skill 目录下的 `profile.md`。
2. 若文件存在且已填写（不是模板占位符），直接用其中的背景，**不要再问用户**。
3. 若 `profile.md` 不存在，提示用户：`cp profile.example.md profile.md` 后填写，今后即可免输入；本次可让用户在对话里直接给背景。
4. 若用户本次在对话里临时给了不同背景，以对话输入为准。

每次仍需用户提供的，只有：
- **导师主页链接**（学校官网 / 实验室主页 / Google Scholar 均可）

profile.md 里应包含：学校+专业+年级、GPA/排名、科研经历（项目/论文/方法）、技能栈、申请方向、其他亮点、姓名、信件语言偏好、申请类别。

### 第 2 步：抓取并分析导师主页
用 WebFetch 读链接，提取：导师**自己措辞**的研究方向、**近 2-3 年代表作**（标题+一句话贡献）、课题组/招生信息、任何能证明"我认真看过"的细节。
主页信息太少时，用 WebSearch 搜"导师姓名 + 单位 + research"补充，或请用户补一个 Scholar/实验室链接。

**导师姓名必须从主页原文核对，绝不凭印象或链接里的拼音猜测：**
1. 从主页正文里找到导师的**全名**和**姓氏**，原样记下来（含正确汉字 / 英文拼写）。
2. 中文名注意：复姓、生僻字、易错同音字都要对准主页写法；英文名注意姓与名的顺序。
3. 主页上同时出现多个名字（同事、合作者、学生）时，确认哪个才是收信人本人。
4. 若主页没有明确姓名或存在歧义，**停下来问用户**，不要先写信。
5. 写信时称呼、正文中出现的导师姓名，全部用这一步核对过的写法，写完后再比对一次。

### 第 3 步：做匹配并先给用户看（最关键）
逐条比对学生背景与导师研究，找 2-3 个**真实、具体**的交集（做过的项目↔导师方向、掌握的方法↔导师论文技术、兴趣↔导师近期具体问题）。
进阶：尝试基于导师论文+领域趋势，提出一个"导师接下来可能关心的问题"的想法。
**先把匹配点和这个想法列给用户确认，再写信。** 宁可承认"无直接经验但有兴趣"，绝不编造。

### 第 4 步：按七段结构生成
按 references/structure.md 写信。

**用 Write 生成一份 markdown 文件交付，文件名 `套磁信-{导师姓名}.md`**（导师姓名用第 2 步核对过的写法）：
- 信件正文正常用 markdown，把核心优势（高水平论文、专业前列排名）用 `**加粗**` 标出来，便于导师一眼扫到。
- 加粗只用在真正的核心优势上，别滥用；其余靠措辞和语序前置。
- 不要在终端回复里再贴一遍全文，交付物就是这份 .md 文件。

**必须提醒用户正确的复制姿势**（否则加粗照样失效）：
> 请用 markdown 预览打开这份文件（VS Code 里按 `Cmd+K V`，或用 Typora），从**渲染后的预览视图**里选中复制，再粘到邮箱的富文本编辑框。直接从文件源码复制会带着 `**` 符号，加粗会失效。

### 第 5 步：强制自检（不可跳过）
写完信后，**必须**逐条对照 references/checklist.md，并在回复里贴出自检结果——每条写明 ✅ 通过 / ❌ 未过，对未过的项说明原因。
- 不允许只说"已自检"或默默跳过；没有贴出逐条结果，就视为没做。
- 任何一条 ❌，先重写对应段落，再重新自检，直到全部 ✅ 才把信交给用户。
- 自检结果展示给用户，让用户也能看到这封信过了哪些关。
